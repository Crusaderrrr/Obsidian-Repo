## Что это такое?
Blockchain - это сеть, состоящая из блоков с информацией, которые связаны между собой специальными ключами. Если изменить какую-то запись и ключи не совпадут - всё разрушится, поэтому информацию в *блоках* можно только просмотреть, но не изменить.

Его идея состоит в том, чтобы без рисков обмениваться чем-либо. Например деньгами, NFT, криптовалютой и тд. Изначально блокчейн использовали для подписания цифровых документов. 

## Как работает Блокчейн

При отправке чего-либо через блокчейн, все его пользователи получат информацию о переводе, потому чтобы мошенникам изменить эту транзакцию им будет необходимо взломать каждое устройство сети, которых могут быть миллионы. Это называется *децентрализацией*. Можно представить себе банк, где у каждого клиента есть копия платежей и переводов каждого из клиентов - это и есть блокчейн. 

## Из чего состоит Блокчейн 

Чтобы соединять звенья цепочки между собой разработчики придумали хитрость - в каждый новый блок добавлять [[Hash and Caching|хеш]] предыдущего. 

Если поменять хеш одного блока, то он не совпадёт с другим и сеть не применит изменения. 

Например, в биткоине все операции хешируются алгоритмом **SHA-256**. Он превращает любую информацию в строку размером 256 бит. Без разницы, будет ли это «Война и мир» Толстого или строчка из песни Михаила Боярского — на выходе получится 256 бит. 

**Пример**: c9f9053e2fb3fcec35ceeafab7bda50ece7d924f886c117b142dfa2df1d63574

## Кто создаёт новые блоки 

Для подбора хеша, создания блоков и в целом для работы блокчейна необходимы большие вычислительные мощности. Эти мощности создают майнеры, за это они получают биткоины, точнее его части.


## Сферы Блокчейна 

- Криптовалюты 
- Смарт-контракты 
	С помощью блокчейна можно заключить безопасный договор с любым контрагентом — например, арендатором квартиры.
- Государственное управление 
	Тайвань использует блокчейн на базе Ethereum для защиты от кибератак со стороны Китая.
- Медицина 
- Интернет вещи 
	Тут блокчейн используется также для безопасности

## На каких языках пишут Блокчейн

Если писать с нуля, то лучше использовать низкоуровневый язык программирования, например *Rust, C++, GO*.

Если необходимо «допилить» блокчейн или написать приложение для него, то подойдут *Python, C#, Java* 

Язык [[Solidity]] был создан специально для блокчейна и работать он будет **только** в блокчейн сети. Его синтаксис был взят из JavaScript.

## Недостатки 

У Блокчейна есть свои уязвимости:
- Атака 51% 
	Чтобы в блокчейне появился новый блок его должны одобрить 50% пользователей. Значит мошенник с железом достаточно мощным может обмахнуть систему и и протолкнуть фальшивые блоки. Но это не выгодно из-за слишком больших затрат.

- Необратимость транзакции 
	Если отправить деньги не туда, вернуть их не получится. 

- Приватные ключи 
	Если потерять свой хеш-ключ, то зайти в свой блокчейн аккаунт не получится и восстановить доступ очен сложно.
