📎 **Сортировка подсчетом** — это алгоритм сортировки набора данных по неубыванию или по невозрастанию. Основным его назначением является сортировка неотрицательных целых чисел, находящихся в небольшом диапазоне.

✔️ Принцип работы алгоритма сортировки подсчетом можно разделить на два этапа:

➖На первом этапе алгоритм определяет, сколько раз каждый элемент сортируемого набора встречается в нем. Для этого алгоритм использует **вспомогательный список**, изначально заполненный нулями: 
`
[0, 0, 0, 0, 0, ...]`

Во вспомогательном списке элемент на позиции с индексом `i` описывает, сколько раз число `i` встречается в сортируемом наборе. Чтобы учесть все числа, которые могут встретиться в наборе, длина вспомогательного списка всегда берется на единицу больше **максимального элемента** сортируемого набора.

Алгоритм проходит по сортируемому набору, выполняет подсчет элементов и заполняет вспомогательный список полученными результатами. Например, если вспомогательный список принимает следующий вид: 
`
[3, 4, 0, 2, 1, ...]`

то это означает, что число `0` в сортируемом наборе встречается `3` раза, число `1` – `4` раза и так далее.

➖Благодаря тому, что индексы во вспомогательном списке, соответствующие элементам сортируемого набора, уже упорядочены, на втором этапе алгоритм просто проходит по вспомогательному списку и указывает каждый элемент столько раз, сколько он встречается. В результате получается исходный набор данных, отсортированный по неубыванию.

⬆️ **Классическая реализация** сортировки подсчетом — это алгоритм, который сортирует список неотрицательных целых чисел **по неубыванию** путем **его изменения**.

При сортировке набора данных длины `n` алгоритм сортировки подсчетом сначала перебирает все `n` элементов сортируемого набора, чтобы определить количество вхождений каждого из них. Затем на основе полученных результатов алгоритм формирует отсортированный набор. Для этого он перебирает целые числа от `0` до `m` включительно, где `m` — максимальный элемент сортируемого набора, и те числа, которые встречались в наборе, указывает столько раз, сколько они встречались.

📎 Таким образом, **временнáя сложность алгоритма сортировки подсчетом** равна `O(n + m)`, где `n` — длина набора данных, а `m` — величина его максимального элемента.
