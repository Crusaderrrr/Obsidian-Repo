Это тип рекурсивной сортировки, который состоит в том, чтобы выбрать *опорный элемент* и затем разделить список на две части: 
- с элементами бóльшими чем опорный
- Элементами меньшими чем опорный 

Повторить тоже самое для каждой из новых частей списка. 
Пример кода: 
```Python
def quicksort(array):
if len(array) < 2:

return array

else:

Базовый случай: массивы с 0 и 1 элементом

уже "отсортированы"

pivot = array[0] Рекурсивный случай

less = [i for i in array[1:] if i < pivot]

greater = [i for i in array[1:] if i > pivot]

Подмассив всех элементов,

меньших опорного

Подмассив всех элементов,

больших опорного

return quicksort(less) + [pivot] + quicksort(greater)

print quicksort([10, 5, 2, 3])
```

Прикол этой сортировки в том, что её временная сложность в **худшем случае** (когда опорный элемент - первый в списке) - О(n). 
![[IMG_1391.jpeg]]

В **лучшем случае** (когда опорный элемент - средний) - O($log_n$)
![[IMG_1392.jpeg]]

И в **среднем случае** (опорный элемент - случайный) - О($n log_n$), так как это среднее между двумя предыдущими:
O(n) * O($log_n$) = O($nlog_n$)

Из этого следует, что луче всего выбирать случайный элемент как опорный.
