#programming 

**Интерпретатор** — это программа, которая выполняет код построчно, без предварительной компиляции в машинный код. Вместо того чтобы сначала переводить весь код в исполняемый файл (как делает компилятор), интерпретатор анализирует и выполняет каждую строку программы "на лету".

### 🔹 Как работает интерпретатор?

1. Читает строку кода.
2. Анализирует её (лексический, синтаксический, семантический анализ).
3. Выполняет команду сразу же.
4. Переходит к следующей строке.

### 🔹 Компилятор vs. Интерпретатор

|**Компилятор**|**Интерпретатор**|
|---|---|
|Переводит весь код сразу|Выполняет код построчно|
|Создаёт отдельный исполняемый файл|Не создаёт файл, выполняет код сразу|
|Быстрее при запуске программы|Медленнее, так как анализирует каждую строку при запуске|
|Пример: C, C++|Пример: Python, JavaScript|

### 🔹 Примеры интерпретаторов

- **Python** (CPython)
- **JavaScript** (V8, SpiderMonkey)
- **Ruby** (MRI)
- **PHP**

Некоторые языки используют **гибридный подход**: например, Java компилируется в байт-код, а потом выполняется интерпретатором (JVM).
