This is when **programs are executed in overlapped time periods**, not necessarily simultaneous.

The **primary goal** is to increase efficiency. 

Main characteristics:
- Multiple threads (can be independent)
- Managing access to shared resources to avoid conflicts 

**Concurrent programming is distinct from the parallelism**, where all the tasks are running exactly at the same time.

*In summary*, concurrent programming allows a system to handle multiple tasks in overlapping time frames, improving responsiveness and resource utilization while requiring careful programming techniques to handle synchronization and avoid issues such as race conditions or deadlocks.
