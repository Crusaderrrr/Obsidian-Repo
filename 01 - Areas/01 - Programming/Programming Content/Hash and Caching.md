# Хеш (Hash)

**Хеш** — это функция, которая преобразует входные данные (например, строку или файл) в фиксированную длины строку символов или чисел, называемую хеш-значением. Хеш-функции используются для проверки целостности данных, создания уникальных идентификаторов или индексов.
Особенности:
- Хеш-значение обычно короче исходных данных.
- Хорошая хеш-функция должна быть быстрой, давать уникальные результаты для разных входных данных (минимизировать коллизии) и быть *детерминированной* (одинаковый вход = одинаковый выход).
- Пример использования: Проверка, был ли файл изменён (если хеш отличается, данные изменены).
*Быстродействие хеш-таблицы*: О(1), оно постоянное (зачастую)

# Кэширование (Caching)

**Кэширование** — это техника хранения копий часто используемых данных в быстром доступе (например, в оперативной памяти), чтобы ускорить их последующий вызов и уменьшить нагрузку на исходный источник (диск, сеть, база данных).
Особенности:
- Данные сохраняются временно и обновляются или очищаются по определённым правилам (например, LRU — Least Recently Used).
- Используется для оптимизации производительности приложений.
- Примеры: кэш браузера, Redis, меморизация функций.
- Пример использования: Хранение результатов сложных вычислений, чтобы не пересчитывать их заново.

# Связь между хешем и кэшированием

Хеш-функции часто используются в кэшировании для создания ключей, по которым данные хранятся и извлекаются. 
Например:
- В хеш-таблицах (hash tables) кэш использует хеш от входных данных (например, строки запроса) как ключ для быстрого доступа к значениям.
- Если входные данные изменились (хеш отличается), кэш понимает, что нужно обновить или перевычислить данные.
- Это ускоряет поиск и проверку в кэше, минимизируя коллизии и обеспечивая целостность.

# Итог
Хеш — инструмент для создания уникальных идентификаторов данных.
Кэширование — способ ускорения доступа к данным.
**Связь**: Хеш помогает эффективно организовать и проверять данные в кэше.
