# Стек
У нас есть функция:
```python
def greet(name):
	print 'hello' + name + '!'
	greet2(name)
	print 'getting ready to say bye...'
	bye()
```
в этой функции есть две другие: `greet2()` и `bye()`
*Стек вызовов* (без функции `print`) это функции будет следующий:
- `greet()`
- `greet2()`
- `greet()`
- `bye()`
- `greet()`
Основная функция это `greet()` она будет в самом низу *стека* и вложенные функции как `greet2()` и `bye()` будут добавляться наверх нашего стека и выполняться, затем они убираются со стека и остаётся только `greet()` потом наверх добавляется следующая функция, выполняется, убирается и так до момента пока функция `greet()` не закончится.

# Рекурсия
Это процесс при котором функция вызывает саму себя.
Например:
```python
def frac(x):
	if x == 1:
		return 1
	else:
		return x * frac(x - 1)
```

Это работает следующим образом:
- Вниз *стека* добавляется основная функция (она не заканчивается, если `x` не равен 1)
- Наверх добавляется следующий вызов функции, только теперь `frax(x - 1)` (она тоже не заканчивается)
- После происходит тоже самое до того момента как один из случаев не выполнится и *стек* не начнёт выполняться сверху вниз (с конца)

Для правильной работы рекурсии необходимы два случая:
- **Базовый** (`if x == 1:`)
- **Рекурсивный** (`else: return x * frac(x - 1)`)

Если это условие не соблюсти, то стек начнёт заполняться бесконечно, пока не кончится память и не выскочит ошибка.

>[!info] **Базовый случай** для массива будет массив из одного элемента или пустой массив (зачастую)
