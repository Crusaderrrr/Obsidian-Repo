# Characteristics 
1. **Statically Typed with Type Inference**  
	Kotlin is statically typed, meaning type checking occurs at compile time, which reduces runtime errors. However, it also supports type inference, so developers rarely need to explicitly declare variable types
2. **Null Safety**
	Exist operators like `?` and `!!` (non-null assertion). No NullPointerException occurs.
3. **Concise and Expressive Syntax**
4. **Object-Oriented and Functional Programming Support**
	Kotlin merges object-oriented principles (classes, inheritance, interfaces) with functional programming constructs such as *lambda expressions*, *higher-order functions*, and *immutable collections*, allowing more modular and declarative coding styles.
5. **Extension Functions**
	Kotlin enables developers to add new functions to existing classes without modifying their source code or using inheritance
6. **Coroutines for Asynchronous Programming**
	Kotlin provides *lightweight [[Coroutine|coroutines]]* natively, making concurrent programming simple and efficient. This feature enables writing asynchronous code without traditional callback complexity, especially in Android and backend development.
7. **Interoperability with Java**
	Kotlin is fully *compatible with existing Java* codebases — developers can call Java code from Kotlin and vice versa. This compatibility has led to its rapid adoption in enterprise and Android projects.
8. **Multiplatform Development**
	Kotlin *compiles not only to the JVM but also to JavaScript* and native binaries (via LLVM), making it a cross-platform language for Android, iOS, web, and desktop applications.
