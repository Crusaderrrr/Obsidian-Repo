**Spring**, besides Beans created be user also loads built-in Beans even if you don't use them. And this process takes very long time. 

**Spring Boot** is the same thing as a regular Spring and has some more features. 
- but loads only Beans that are used in the application, even if these are built-in. 
- To enable built-in Bean use a keyword. 
- There is a built-in Web Server/Application Server *Tomcat*, that can receive and send requests and responses.
