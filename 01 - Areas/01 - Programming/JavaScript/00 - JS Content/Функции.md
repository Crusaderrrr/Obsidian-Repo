### Closures
- **Functions** that "*remember*" the variables in their outer scope even after the outer function has finished executing.
```JavaScript
function createCounter() { 
	let count = 0; 
	return function() {}
		count++; 
		return count; 
	}; 
}

const counter = createCounter(); 
console.log(counter()); // 1 
console.log(counter()); // 2 (count persists due to closure)
```

###  Function declaration and expression

**Function declaration**:
  ```javascript
sayHi("Вася"); // Работает 

function sayHi(name) { 
	console.log("Привет, " + name); 
}
```
- This function is *hoisted* and can be accessed before it is declared.

**Function expression**:
```javascript
sayHi("Вася"); // Ошибка! 

const sayHi = function(name) { 
	console.log("Привет, " + name); 
};
``` 
- This function is *not hoisted*, and can not be  accessed before it is declared.
- Function Expression часто используют для создания анонимных функций, передачи функций как аргументов, а также для условного объявления функций.
