*Паттерны проектирования* — это шаблоны решений типовых задач проектирования программ. 
 
Используются для программ, написанных на объектно-ориентированных языках. Типовые задачи проектирования таких программ можно разделить на три основные категории:

- создание новых объектов
- структурирование системы на основе классов и объектов
- построение связей между объектами и распределение обязанностей между ними

Паттерны проектирования также подразделяются на категории в соответствии с задачами, которые они решают:

- **Порождающие** — предназначены для создания объектов, позволяют системе оставаться независимой как от самого процесса создания, так и от типов создаваемых объектов
- **Структурные** — рассматривают вопросы грамотной компоновки частей системы
- **Поведенческие** — решают проблемы взаимосвязи различных частей системы

Существуют для упрощения кода и для того чтобы не создавать велосипед.
